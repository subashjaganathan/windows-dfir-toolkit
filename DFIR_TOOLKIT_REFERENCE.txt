1. OVERVIEW
================================================================================

The Windows DFIR Toolkit is a PowerShell-based forensic evidence collection
suite designed for incident response investigations on Windows endpoints.

One command collects all forensic evidence from a Windows machine and produces:
  - Structured JSON evidence files with chain of custody
  - SHA256 integrity hash per evidence file
  - Professional HTML incident response report
  - Unified forensic timeline (HTML + CSV)
  - IOC extraction with VirusTotal enrichment
  - Evidence manifest with full chain of custody

Key Features:
  - 62 scripts across 24 categories (61 collection + AI attack detection)
  - Auto-adapts collection for Workstation vs Server OS
  - Read-only forensic-safe collection
  - Full chain of custody with NTP time verification
  - NIST 800-61, NIST 800-86, ISO 27037 compliant
  - Run all scripts OR individual scripts OR phases
  - Auto-downloads WinPmem for live RAM capture

2. REQUIREMENTS
================================================================================

MINIMUM REQUIREMENTS:
  - Windows 10 (build 1607+), Windows 11, or Windows Server 2016/2019/2022
  - PowerShell 5.1 (built-in on all supported OS versions)
  - Administrator privileges
  - 2 GB free disk space (10+ GB recommended for full collection with RAM dump)

RECOMMENDED:
  - Run as Domain Admin or local Administrator
  - Disable or exclude toolkit folder from Windows Defender before running
  - Internet access for WinPmem auto-download (RAM dump only)
  - VirusTotal API key for IOC enrichment (free at virustotal.com)

OPTIONAL TOOLS (place in Tools\ folder):
  - winpmem_mini_x64.exe  : Live RAM capture (auto-downloaded if not present)
                            Download: github.com/Velocidex/WinPmem/releases

NO ADDITIONAL SOFTWARE REQUIRED for all other 59 scripts.

3. INSTALLATION
================================================================================

STEP 1 - Download
  Download windows-dfir-toolkit-v1.0.zip from your source

STEP 2 - Extract
  Extract to a local folder (avoid network drives for performance)
  Recommended: C:\DFIR\ or Desktop

  The extracted structure should be:
    windows-dfir-toolkit-v1.0\
      Run_IR_Collection.ps1     <- Master runner
      Scripts\                  <- All 44 collection scripts
      Tools\                    <- Place winpmem here
      DFIR_TOOLKIT_REFERENCE.txt

STEP 3 - Add AV Exclusion (Recommended)
  Open PowerShell as Administrator and run:

    Add-MpPreference -ExclusionPath "C:\path\to\windows-dfir-toolkit-v1.0"

  Remove after collection:
    Remove-MpPreference -ExclusionPath "C:\path\to\windows-dfir-toolkit-v1.0"

STEP 4 - Set Execution Policy
  Set-ExecutionPolicy Bypass -Scope Process -Force

4. QUICK START
================================================================================

FULL COLLECTION (Recommended):

  Step 1: Open PowerShell as Administrator
          Right-click PowerShell -> Run as Administrator

  Step 2: Navigate to toolkit
          cd "C:\path\to\windows-dfir-toolkit-v1.0"

  Step 3: Add AV exclusion
          Add-MpPreference -ExclusionPath (Get-Location).Path

  Step 4: Set case details
          $env:DFIR_CASE = "IR-2026-001"
          $env:DFIR_INV  = "InvestigatorName"
          $env:DFIR_DAYS = "30"

  Step 5: Allow execution
          Set-ExecutionPolicy Bypass -Scope Process -Force

  Step 6: Run
          .\Run_IR_Collection.ps1

  Step 7: View results
          explorer C:\IR_Collection

EXPECTED RUNTIME: 15-30 minutes depending on system size

7. OUTPUT STRUCTURE
================================================================================

All output is saved to C:\IR_Collection\

  C:\IR_Collection\
  |
  +-- System_Info_HOSTNAME_TIMESTAMP.json
  +-- System_Info_HOSTNAME_TIMESTAMP.json.hash.json
  |
  +-- Network_Connections_HOSTNAME_TIMESTAMP.json
  +-- Network_Connections_HOSTNAME_TIMESTAMP.json.hash.json
  |
  +-- Security_EventLog_HOSTNAME_TIMESTAMP.json
  +-- ... (one .json + one .hash.json per script)
  |
  +-- Prefetch_HOSTNAME_TIMESTAMP\        <- Prefetch .pf files
  +-- EventLogs_Raw_HOSTNAME_TIMESTAMP\   <- Raw .evtx files
  +-- Browser_Artifacts_HOSTNAME_TIMESTAMP\ <- SQLite DB files
  +-- MFT_USN_HOSTNAME_TIMESTAMP\         <- MFT and USN files
  +-- RAM_HOSTNAME_TIMESTAMP\             <- Memory dump .raw
  |
  +-- Report_TIMESTAMP\
  |     +-- IR_Report_HOSTNAME_TIMESTAMP.html    <- Main HTML report
  |     +-- IR_Summary_HOSTNAME_TIMESTAMP.txt    <- Text summary
  |     +-- IOC_HOSTNAME_TIMESTAMP.json          <- IOC export
  |     +-- IOC_HOSTNAME_TIMESTAMP.csv           <- SIEM-ready CSV
  |
  +-- Timeline_TIMESTAMP\
  |     +-- Timeline_HOSTNAME_TIMESTAMP.html     <- Searchable timeline
  |     +-- Timeline_HOSTNAME_TIMESTAMP.csv      <- Timeline CSV export
  |
  +-- IOC_ThreatIntel_HOSTNAME_TIMESTAMP.html    <- VirusTotal results
  |
  +-- Evidence_Manifest_TIMESTAMP.json           <- Master index + hashes
  +-- Evidence_Manifest_TIMESTAMP.json.hash.json
  +-- MasterRun_TIMESTAMP.log                    <- Full execution log
  +-- *_Execution.log                            <- Per-script audit logs

FILE TYPES:
  *.json           - Structured evidence with chain of custody envelope
  *.hash.json      - SHA256 integrity proof for each evidence file
  Evidence_Manifest - Master index of all files with hashes and timestamps
  MasterRun.log    - Complete execution log with timings and errors
  IR_Report.html   - Professional HTML incident response report
  Timeline.html    - Interactive searchable forensic event timeline
  IOC*.csv         - SIEM-ready IOC file for Splunk/Sentinel/Elastic import

12. TROUBLESHOOTING
================================================================================

KNOWN ISSUES AND WORKAROUNDS:

  Issue   : SRUM_PowerShell_History blocked by Defender
  Cause   : Defender heuristic fires before script starts
  Fix     : Run before toolkit:
              Add-MpPreference -ExclusionPath "C:\path\to\toolkit"
              Add-MpPreference -ExclusionProcess "powershell.exe"

  Issue   : MFT 0/N volumes collected
  Cause   : No VSS on machine or Group Policy blocks backup privilege
  Status  : robocopy /B attempted automatically - metadata always collected
  Manual  : FTK Imager -> Add Evidence -> Logical Drive for full MFT

  Issue   : PCAP file 0 MB
  Cause   : Earlier versions passed provider= switches to netsh, which
            disables packet capture. Also, some Windows builds/VMs disable
            NDIS packet capture at the OS level.
  Status  : FIXED - capture now uses "netsh trace start capture=yes" with no
            providers (correct packet-capture form) plus a Running-status check.
  Manual  : If still 0 MB, the host's netsh packet capture is OS-disabled;
            use Wireshark/tshark for guaranteed capture.

  Issue   : Timeline 0 events
  Cause   : $Host is a reserved PowerShell variable; using it as an Add-Event
            parameter threw on every call and the outer try/catch hid it.
  Status  : FIXED - parameter renamed to $HostVal, silent catch removed,
            plus DateTimeOffset.Parse (InvariantCulture) for +05:30 offsets.
            Now produces full timeline (3000+ events on a typical host).

  Issue   : PowerShell_EventLog OutOfMemoryException
  Cause   : Two factors on heavily-logged hosts: loading all 4104 events at
            once, and ConvertTo-Json -Depth 8 expanding every object 8 levels.
  Status  : FIXED - events streamed with a hard cap; JSON now serialized at
            -Depth 5 -Compress with a suspicious-only fallback if volume is
            extreme. Full script text remains in the raw EVTX export.
